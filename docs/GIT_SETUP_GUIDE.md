## ğŸ”¢ **PART 4: Version Numbering**

### **Semantic Versioning: MAJOR.MINOR.PATCH**

**Format:** `0.1.0`, `1.2.3`, `2.0.0`

---

### **PATCH Version (x.x.PATCH)**

**Increment when:**
- Bug fixes
- Documentation updates
- Small improvements
- No new features
- No breaking changes

**Examples:**
```
0.1.0 â†’ 0.1.1: Fixed AI memory bug
0.1.1 â†’ 0.1.2: Fixed UI scrolling
0.1.2 â†’ 0.1.3: Updated documentation
```

---

### **MINOR Version (x.MINOR.x)**

**Increment when:**
- New features added
- Completing a Phase
- Significant functionality
- Backward compatible changes

**Examples:**
```
0.1.3 â†’ 0.2.0: Phase 2 complete (Intelligence & Autonomy)
0.2.0 â†’ 0.3.0: Phase 3 complete (Communication Services)
0.3.0 â†’ 0.4.0: Phase 4 complete (Advanced Features)
```

**Reset PATCH to 0:**
```
0.1.5 â†’ 0.2.0 (not 0.2.5)
```

---

### **MAJOR Version (MAJOR.x.x)**

**Increment when:**
- Breaking changes
- Complete rewrite
- Incompatible with previous versions
- Major milestone

**Examples:**
```
0.6.0 â†’ 1.0.0: All 56 features complete, production ready
1.5.0 â†’ 2.0.0: Complete architecture redesign
```

**Reset MINOR and PATCH:**
```
0.6.5 â†’ 1.0.0 (not 1.6.5)
```

## ğŸ“ **PART 5: Commit Messages**

### **Good Commit Message Format**

```
type: Short description (50 chars max)

- Detailed bullet points explaining:
  - What changed
  - Why it changed
  - What it fixes or adds

Refs: #issue_number (if applicable)
```

---

### **Commit Types**

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation only
- `style:` Formatting, no code change
- `refactor:` Code restructuring, no behavior change
- `test:` Adding or updating tests
- `chore:` Maintenance, build changes
- `perf:` Performance improvements

---

### **Examples**

**Good Commit Messages:**

```bash
git commit -m "fix: AI now remembers conversation history

- Added conversation context to system prompt
- Loads last 50 messages from database
- Includes messages in chronological order

Fixes #3"
```

```bash
git commit -m "feat: add file upload with PDF support

- Implemented FileUploadHandler class
- Supports PDF, DOCX, TXT, images
- OCR for image text extraction
- API endpoints: /api/upload, /api/uploads

Refs: Phase 3 requirements"
```

```bash
git commit -m "docs: add SOUL_OF_THE_PROJECT.md

- Emotional continuity document
- Explains project philosophy
- For future Claude instances
- Preserves collaborative spirit"
```

**Bad Commit Messages:**

```bash
git commit -m "fixed stuff"          # Too vague
git commit -m "asdf"                 # Meaningless
git commit -m "Updated files"        # Not specific
git commit -m "WIP"                  # Not descriptive
```

## ğŸš¨ **PART 12: Common Issues**

### **Issue: "Already exists" when pushing**

```bash
# Pull first
git pull origin main --rebase

# Then push
git push origin main
```

---

### **Issue: Accidentally committed large files**

```bash
# Remove from Git but keep file
git rm --cached data/databases/evolution.db

# Add to .gitignore
echo "data/databases/*.db" >> .gitignore

# Commit
git add .gitignore
git commit -m "chore: stop tracking database files"
```

---

### **Issue: Want to undo last commit**

```bash
# Undo commit but keep changes
git reset --soft HEAD~1

# Undo commit and discard changes (careful!)
git reset --hard HEAD~1
```

---

### **Issue: Wrong commit message**

```bash
# If haven't pushed yet
git commit --amend -m "New message"

# If already pushed (avoid this)
git commit --amend
git push --force origin main  # Dangerous!
```
